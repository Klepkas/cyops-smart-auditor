**Scan engine design (`src/lib/scanEngine.ts`)**
- Input: `code: string`, `progress: (state: AgentState) => void`, `signal: AbortSignal`.
- Pipeline: hash code → derive a deterministic seed → run four sequential async "agents" each sleeping `400–900ms` (jittered) and emitting a typed progress event.
- Each agent contributes a typed slice of `Report`:
  - `staticAnalyzer` → `Vulnerability[]` (uses `vulnerabilityPatterns.ts` regex bank for demo realism — reentrancy, tx-origin, unchecked-call, integer-overflow, etc.).
  - `symbolicExecutor` → adds path-coverage findings.
  - `aiReviewer` → `Suggestion[]` (remediation snippets).
  - `gasOptimizer` → `GasTip[]` (e.g., "use `unchecked++` in loops", "pack struct fields", "cache storage reads").
- `riskScore` = weighted sum clamped to `[0, 100]`; severity weights: critical 25, medium 10, low 3.
- All work runs client-side; no network calls.

**Routing & layout**
- `App.tsx` wraps `<BrowserRouter>` around a single `<Layout>` route with nested children: `/`, `/auditor`, `/history`, `/settings`. `/` redirects to `/dashboard`.
- `<Layout>` renders `<Sidebar />` + `<Topbar />` + `<Outlet />`. Topbar holds the page title (route-aware) and a global "New Scan" CTA that routes to `/auditor`.

**Styling rules**
- Tailwind only — no CSS-in-JS, no inline `style={}` except for the radial gauge's `strokeDasharray`.
- Color tokens (locked in `tailwind.config.ts`):
  - `brand`: indigo-violet scale (`#6366f1` → `#a78bfa`)
  - `surface`: `#0b0d12` (page bg), `#11141b` (panels), `#1a1f2b` (elevated)
  - `text`: `#e6e8ef` (primary), `#9aa3b2` (muted)
  - `risk.critical`: `#ef4444`, `risk.medium`: `#f59e0b`, `risk.low`: `#22c55e`, `risk.info`: `#3b82f6`
- Focus ring: `focus-visible:ring-2 focus-visible:ring-brand-500/60 focus-visible:outline-none` on every interactive element.

**State persistence**
- `useScanHistory` writes to `localStorage` key `smart-auditor:history` (JSON array, max 100, FIFO eviction).
- `useSettings` writes to `localStorage` key `smart-auditor:settings`.

**Error handling**
- Editor mount failure → fallback to a styled `<textarea>`.
- Scan engine throws only on `AbortError` (cancel); UI treats it as a normal "Scan cancelled" toast.
- localStorage quota errors are caught and surfaced as a Settings-page banner ("History storage full — older scans were removed").

**Performance**
- CodeMirror bundle is the largest dep; import only the Solidity + line-numbers extensions.
- `useMemo` the report-derived dashboard stats so the trend chart doesn't re-render on every keystroke in the editor.

**Risk controls**
- CodeMirror version drift → pin to `^4` and lock the lang package to a working minor.
- Tailwind v4 surprise → explicitly install v3.4 to avoid the new engine in this iteration.
- Re-render storms when typing → memoize dashboard derivations; the editor uses its own internal state and never re-renders the report.
- Determinism complaints → seed the RNG with a hash of the code so the same input yields the same report; document this in the README.
